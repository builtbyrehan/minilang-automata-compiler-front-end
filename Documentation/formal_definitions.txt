FORMAL DEFINITIONS FOR MINILANG COMPILER FRONT-END
==================================================

Project Title:
MiniLang: Automata-Based Compiler Front-End

Topic Code:
F2 - Automata in Compiler Design


1. DFA FORMAL DEFINITION FOR LEXICAL ANALYZER
---------------------------------------------

A Deterministic Finite Automaton is formally defined as:

DFA = (Q, Σ, δ, q0, F)

Where:

Q  = Finite set of states
Σ  = Input alphabet
δ  = Transition function
q0 = Initial state
F  = Set of final/accepting states


In this project, DFA is used to recognize tokens such as:

1. Keywords
2. Identifiers
3. Numbers
4. Operators
5. Separators


2. DFA FOR IDENTIFIER RECOGNITION
---------------------------------

Language accepted:

Identifier = letter (letter | digit | _)*

Examples accepted:

x
marks
total1
student_marks

Examples rejected:

123abc
@value
5marks


Formal DFA:

Q = {q0, q1, qd}

Σ = {letters, digits, underscore, others}

q0 = Initial state

F = {q1}

Transition function:

δ(q0, letter)     = q1
δ(q0, underscore) = q1
δ(q0, digit)      = qd
δ(q0, other)      = qd

δ(q1, letter)     = q1
δ(q1, digit)      = q1
δ(q1, underscore) = q1
δ(q1, other)      = qd

δ(qd, any input)  = qd


3. DFA FOR NUMBER RECOGNITION
-----------------------------

Language accepted:

Number = digit+

Examples accepted:

10
85
100
2026

Examples rejected:

12abc
x123
@45


Formal DFA:

Q = {q0, q1, qd}

Σ = {digits, letters, underscore, others}

q0 = Initial state

F = {q1}

Transition function:

δ(q0, digit)      = q1
δ(q0, letter)     = qd
δ(q0, underscore) = qd
δ(q0, other)      = qd

δ(q1, digit)      = q1
δ(q1, letter)     = qd
δ(q1, underscore) = qd
δ(q1, other)      = qd

δ(qd, any input)  = qd


4. CONTEXT-FREE GRAMMAR FORMAL DEFINITION
-----------------------------------------

A Context-Free Grammar is formally defined as:

CFG = (V, Σ, P, S)

Where:

V = Set of non-terminals
Σ = Set of terminals
P = Set of production rules
S = Start symbol


For MiniLang:

V = {
Program,
StatementList,
Statement,
Declaration,
Assignment,
PrintStatement,
IfStatement,
Condition,
Expression,
RelOp,
ArithOp
}


Σ = {
int,
if,
print,
id,
number,
=,
+,
-,
*,
/,
>,
<,
;,
(,
),
{,
}
}


S = Program


5. MINILANG PRODUCTION RULES
----------------------------

Program → StatementList

StatementList → Statement StatementList | ε

Statement → Declaration
          | Assignment
          | PrintStatement
          | IfStatement

Declaration → int id = number ;

Assignment → id = Expression ;

PrintStatement → print ( id ) ;

IfStatement → if Condition { StatementList }

Condition → id RelOp number

Expression → id
           | number
           | id ArithOp number
           | number ArithOp number
           | id ArithOp id
           | number ArithOp id

RelOp → > | <

ArithOp → + | - | * | /


6. RELATION BETWEEN DFA AND CFG IN THIS PROJECT
-----------------------------------------------

The MiniLang compiler front-end uses two computational models:

1. DFA:
   Used in lexical analysis to recognize valid tokens.

2. CFG:
   Used in syntax analysis to check whether the sequence of tokens follows valid grammar rules.

Therefore, the project integrates regular language recognition and context-free grammar parsing in a compiler design application.
