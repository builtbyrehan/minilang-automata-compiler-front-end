MINILANG COMPUTATION TRACES
===========================

Project Title:
MiniLang: Automata-Based Compiler Front-End

Topic:
F2 - Automata in Compiler Design


1. PURPOSE OF COMPUTATION TRACES
--------------------------------

Computation traces show how the MiniLang compiler front-end processes source code step by step.

Each trace explains:

1. Source code input
2. Lexical analysis
3. Token stream generation
4. Syntax analysis
5. Final result


2. TRACE 1: VALID DECLARATION AND PRINT STATEMENT
-------------------------------------------------

Input Code:

int x = 10;
print(x);


Step 1: Lexical Analysis

The lexical analyzer scans the input code character by character.

Generated Tokens:

KEYWORD(int)
IDENTIFIER(x)
OPERATOR(=)
NUMBER(10)
SEMICOLON(;)
KEYWORD(print)
LPAREN(()
IDENTIFIER(x)
RPAREN())
SEMICOLON(;)


Step 2: Syntax Analysis

The parser checks the token stream using CFG rules.

Matched Grammar Rules:

Program → StatementList

StatementList → Statement StatementList

Statement → Declaration

Declaration → int id = number ;

Statement → PrintStatement

PrintStatement → print ( id ) ;


Step 3: Final Result

Lexical Analysis: Passed
Syntax Analysis: Passed
Result: Valid MiniLang Code


3. TRACE 2: VALID IF STATEMENT
------------------------------

Input Code:

int marks = 85;
if marks > 50 {
    print(marks);
}


Step 1: Lexical Analysis

Generated Tokens:

KEYWORD(int)
IDENTIFIER(marks)
OPERATOR(=)
NUMBER(85)
SEMICOLON(;)
KEYWORD(if)
IDENTIFIER(marks)
OPERATOR(>)
NUMBER(50)
LBRACE({)
KEYWORD(print)
LPAREN(()
IDENTIFIER(marks)
RPAREN())
SEMICOLON(;)
RBRACE(})


Step 2: Syntax Analysis

Matched Grammar Rules:

Program → StatementList

StatementList → Statement StatementList

Statement → Declaration

Declaration → int id = number ;

Statement → IfStatement

IfStatement → if Condition { StatementList }

Condition → id RelOp number

RelOp → >

StatementList → Statement StatementList

Statement → PrintStatement

PrintStatement → print ( id ) ;


Step 3: Final Result

Lexical Analysis: Passed
Syntax Analysis: Passed
Result: Valid MiniLang Code


4. TRACE 3: INVALID CODE WITH MISSING SEMICOLON
-----------------------------------------------

Input Code:

int x = 10
print(x);


Step 1: Lexical Analysis

Generated Tokens:

KEYWORD(int)
IDENTIFIER(x)
OPERATOR(=)
NUMBER(10)
KEYWORD(print)
LPAREN(()
IDENTIFIER(x)
RPAREN())
SEMICOLON(;)


Step 2: Syntax Analysis

The parser tries to match:

Declaration → int id = number ;

The parser successfully matches:

int
id
=
number

But after number, it expects:

SEMICOLON(;)

Instead, it finds:

KEYWORD(print)


Step 3: Error Report

Syntax Error:
Expected semicolon after declaration.


Step 4: Final Result

Lexical Analysis: Passed
Syntax Analysis: Failed
Result: Invalid MiniLang Code

7. SUMMARY OF TRACES
--------------------

The computation traces show that the MiniLang compiler front-end can:

1. Generate tokens from source code
2. Recognize valid declaration statements
3. Recognize valid print statements
4. Recognize valid if statements
5. Detect missing semicolons
6. Detect invalid statement structures
7. Report syntax errors with meaningful messages

These traces demonstrate the integration of DFA-based lexical analysis and CFG-based syntax analysis in compiler design.
